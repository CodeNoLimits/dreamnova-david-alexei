The Golden Rules

1
Read before you write. Always read the file before editing it. Never assume project state.
2
Conventional commits only. feat: fix: chore: — one feature per commit.
3
Lighthouse > 90. Performance, accessibility, SEO. Test before every deploy.

Never revoke API keys. If something breaks, debug. Never delete credentials.
